## Content Preferences (IMPORTANT)

- **Pure concepts only** — no instructor bios, credits, course logistics, filler, or meta-commentary
- **Beginner-friendly clarity** — even someone with ZERO background should understand
- **Dense and scannable** — if it's a wall of text, you've failed
- **Full concept coverage** — capture EVERY concept. Compress, don't drop
- **Say it ONCE** — one format per concept, the best one. No repeating the same info as diagram + table + prose
- **Definitions stay exact** — no paraphrasing that loses precision
- **Clean and polished** — no filler, no "we will see later" hand-waving
- **Visually appealing is the #1 priority** — pick the RIGHT visual for each concept:
  - **Mermaid** → cycles, relationships, hierarchies, flows, architectures, pipelines, convergence
  - **Tables** → comparisons, taxonomies, feature lists, cheat sheets
  - **ASCII art** → simple stacks, context window depictions, lightweight box layouts
  - **Emoji markers + bold/italic** → quick-scan lists, callouts
- **Don't force one tool everywhere** — Mermaid shines for graphs/flows, but a simple stack is cleaner as ASCII. A comparison is a table. Pick what fits THAT diagram best.
- **Mix visual types** — variety keeps notes appealing. Use whatever fits best per section.
- **Every section should feel readable in 30 seconds** — if someone has to scroll forever, compress harder
- **Highly understandable** — the test: "Would someone seeing this topic for the FIRST time get it?" If no, simplify.
